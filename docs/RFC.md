# System design

Сервис для PayMeMore — оценивание зарплаты по резюме

- Пользователь
- Резюме
- Вакансии

## 1. Требования

- Обязательные функциональные требования
  - Пользователь может рассчитать вилку ЗП по:
    - Роли
    - Опыту
    - Навыкам
    - Региону
  - Пользователь может получить по рекомендации по улучшению резюме от LLM
  - Пользователь может получить подходящие ему вакансии (из датасета)
  
- Дополнительные функциональные требования
  - Система сохраняет результаты поиска вилки ЗП и рекомендаций, чтобы пользователь мог просмотреть это
  - Пользователь может импортировать резюме из различных источников (PDF, HH.RU)
  - Пользователь может экспортировать свое резюме в PDF
  - Пользователь может увидеть оценку резюме по мнению LLM: плохо/нормально/хорошо
  - Пользователь может получить подходящие ему вакансии (с HH.RU)
- Опциональные функциональные требования
  - "Прожарка" резюме от ИИ
  - Пользователь может добавить вакансию в избранное
  - Пользователь может сохранить список рекомендуемых вакансий как PDF-файл
  - После заполнения резюме система предлагает принять ИИ-правки. Пользователь может нажать на кнопку "Применить рекомендации" и его резюме автоматически обновится в соответствии с рекомендациями от ИИ.
  - Пользователи могут зайти на лендинг, на котором есть скриншоты интерфейса и описанием основной функциональности, чтобы было легче начать пользоваться платформой.

- Нефункциональные требования
	- MVP
		- DAU: XXX пользователей
		- RPS 
    - SLA: 90% (Возможный downtime 36 дней в год)
