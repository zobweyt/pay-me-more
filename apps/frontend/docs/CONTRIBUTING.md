# Руководство по разработке фронтенда

**Это руководство именно по разработке фронтенда, которое не затрагивает общие принципы, бэкенд или развёртывание.**

Подразумевается, что вы уже прочитали [общее руководство по разработке](../../../docs/CONTRIBUTING.md).

## Предварительные требования

- [Node.js](https://nodejs.org) 24 или выше.

## Настройка окружения

1. Перейдите в директорию приложения:

   ```sh
   cd apps/frontend
   ```

2. Установите зависимости с git-хуками для соблюдения код-стайла:

   ```sh
   npm i
   ```

> [!NOTE]
> В случае необходимости вы можете не выполнять git-хуки перед коммитом при помощи аргумента [--no-verify](https://git-scm.com/docs/githooks#_pre_commit).

## Запуск проекта

Для запуска приложения в режиме разработки используйте:

```sh
docker compose --profile dev up
```

Альтернативно, можно запустить проект вместе с HTTPS туннелем через [CloudPub](https://cloudpub.ru/):

```sh
docker compose --profile dev-https up
```

После запуска будут доступны следующие ссылки:

- Основное фронтенд приложение: http://127.0.0.1.
- Интерактивная документация API: http://127.0.0.1/api/docs.
- Админ панель: http://127.0.0.1/api/admin.

## Основные скрипты

- Форматирование кода:

  ```sh
  npm run format
  ```

- Анализ кода на потенциальные ошибки:

  ```sh
  npm run lint
  ```

- Проверка форматирования и анализ кода:

  ```sh
  npm run check
  ```

- Проверка типизации:

  ```sh
  npm run tsc
  ```

- Обновление типизации API (перед выполнением команды ниже убедитесь, что [проект запущен в режиме разработки](#запуск-проекта)):

  ```sh
  npm run openapi
  ```

## Очистка окружения

Для остановки и удаления контейнеров, включая volumes, выполните:

```sh
docker compose --profile dev down -v
```

Удаление установленных зависимостей:

```sh
rm -rf node_modules
```
